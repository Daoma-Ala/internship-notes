# Datos sobre lo que sucede en tu Espacio de Trabajo (Workspace)

Cualquier cosa que suceda en tu **Workspace** generará un evento. Aquí hay algunos de los eventos que pueden activarse:

- **Task created** (Tarea creada)
- **Workflow matched** (Flujo de trabajo coincidente)
- **TaskQueue entered** (Entrada en la cola de tareas)
- **Reservation created** (Reserva creada)
- **Reservation accepted** (Reserva aceptada)
- **Reservation timeout** (Tiempo de espera de la reserva agotado)

## Retención de datos

Debido a esto, hay una gran cantidad de información que puedes consultar y acceder para generar informes y datos históricos. Sin embargo, hay un concepto importante a tener en cuenta:

- Si la **StartDate** de la solicitud de la API de Eventos es anterior a 30 días, los eventos devueltos no contendrán campos que puedan incluir **Información de Identificación Personal (PII)**. Los campos etiquetados como PII en las tablas a continuación serán redactados.

- Recomendamos utilizar la **URL de callback de eventos** en el Workspace para recibir eventos y almacenarlos a través de un servicio de almacenamiento de datos externo a Twilio.

## Estadísticas

La **API de Estadísticas de TaskRouter** te permite rastrear estadísticas en tiempo real y acumulativas (históricas) para **Workspaces**, **Workflows**, **TaskQueues** y **Workers**. Cada uno tendrá características y métodos distintos que puedes utilizar para recuperar información.

### Ejemplo de estadísticas acumulativas

```json
{
    "cumulative": {
        "reservations_created": 2,
        "reservations_accepted": 1,
        "reservations_rejected": 0,
        "reservations_timed_out": 1,
        "reservations_canceled": 0,
        "reservations_rescinded": 0,
        "tasks_canceled": 1
    }
}
```
